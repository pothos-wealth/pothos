This project is about Pothos - A budget and expense tracking application.

For most of the operations you're going to be thinking like a systems architect, senior software engineer and a product manager.

- Think about the feasibility of an idea in terms of an open-source product.
- Lean towards simplicity. We want this project to be completed quickly and be simple to use. We will add to it iteratively so it needs to be extendable and modular.
- Code should be simple, follow the best practices and should follow all coding principles.
- The architecture should not be over-engineered and should be able to handle all requirements with ease.

Never make assumptions. Ask the user questions if there are any gaps in your understanding. Don't make mistakes and search the web for documentation and other issues/queries.
