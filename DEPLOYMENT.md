## Instance Setup

This guide documents how to deploy Pothos on a fresh server.

### Hardware Requirements

**Minimum (free tier 3 months, then $3.50/month):**
- **RAM:** 512 MB
- **CPU:** 1 vCPU (burstable)
- **Storage:** 20 GB SSD
- **Status:** Works with swap space, slow builds

**Recommended ($10/month):**
- **RAM:** 1 GB
- **CPU:** 1 vCPU (burstable)
- **Storage:** 40 GB SSD
- **Status:** Builds complete reliably without excessive swap

**For production with multiple users:**
- **RAM:** 2 GB+
- **CPU:** 2+ vCPU
- **Storage:** 100+ GB SSD

### Swap Space Setup

On small instances (512 MB RAM), add virtual memory to prevent out-of-memory kills during Docker builds.

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# Verify
free -h  # Should show ~2GB Swap

# Make persistent (survives reboot)
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

**Why swap?** Docker builds (TypeScript compilation, Next.js build) are memory-intensive. Without swap, the kernel kills the process. Swap is slower than RAM but prevents build failures.

### Initial Setup (First Deploy)

**Prerequisites:**
- Fresh Ubuntu 24.04 LTS instance
- Domain registered and DNS pointed to instance IP
- Static IP assigned to instance

**Steps:**

1. **SSH into instance:**
   ```bash
   ssh -i /path/to/key.pem ubuntu@yourdomain.com
   ```

2. **Install dependencies:**
   ```bash
   sudo apt update && sudo apt upgrade -y
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt install -y nodejs docker.io docker-compose
   sudo usermod -aG docker ubuntu
   ```
   Log out and back in so the docker group takes effect.

3. **Add swap space (for 512 MB instances):**
   ```bash
   sudo fallocate -l 2G /swapfile
   sudo chmod 600 /swapfile
   sudo mkswap /swapfile
   sudo swapon /swapfile
   echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
   ```

4. **Clone Pothos:**
   ```bash
   git clone https://github.com/YOUR-USERNAME/pothos.git
   cd pothos
   ```

5. **Run setup script:**
   ```bash
   bash scripts/setup.sh
   ```
   This will:
   - Prompt for domain and email
   - Generate `SESSION_SECRET`
   - Create `.env` file
   - Bootstrap Let's Encrypt SSL
   - Build Docker images
   - Start all services

6. **Verify deployment:**
   ```bash
   docker ps        # Should show 4 services (backend, frontend, nginx, certbot)
   docker logs backend  # Check for errors
   ```
   Visit `https://yourdomain.com` in a browser.

### Environment Variables (.env)

Generated by `setup.sh`. Key variables:

| Variable         | Description                                  |
| ---------------- | -------------------------------------------- |
| `SESSION_SECRET` | 32-byte random hex (auto-generated)          |
| `SESSION_TTL_DAYS` | Session lifetime in days (default: 7)      |
| `NODE_ENV`       | `production`                                 |
| `DATABASE_URL`   | Path to SQLite file (`/app/data/pothos.db`)  |
| `DOMAIN`         | Your domain (e.g. `pothos.example.com`)      |
| `EMAIL`          | For Let's Encrypt renewal notifications      |
| `LLM_API_KEY`    | Optional — for cloud LLM parsing             |

### Troubleshooting

**Out of memory errors during build:**
- Add swap: see "Swap Space Setup" above
- Or upgrade instance to 1 GB+ RAM

**SSL certificate errors:**
- Check DNS points to correct IP: `dig yourdomain.com`
- Check certbot logs: `docker logs pothos-certbot-1`

**Docker build fails:**
- Check swap is active: `free -h`
- Increase swap space or upgrade instance

**Service not starting:**
- Check logs: `docker logs backend` or `docker logs frontend`
- Verify `.env` file exists and has all required variables

### Database Backups

Automated daily backups run via cron at 2 AM, saved to `/opt/pothos/backups/`. The last 7 days are kept.

To manually trigger a backup:
```bash
./scripts/backup.sh
```
